Concepto Descripción Concepto Descripción
Tabla en html Una tabla en HTML es un elemento que organiza y presenta datos en un formato de filas y columnas, facilitando la visualización y comparación de información. Se define mediante la etiqueta (table) y se compone de filas (tr), celdas de encabezado (th) y celdas de datos (td). Formulario en html Un formulario en HTML es un elemento que permite a los usuarios enviar datos a un servidor mediante la interacción con diversos campos de entrada, como texto, botones, casillas de verificación y menús desplegables. Se define con la etiqueta (form) y puede incluir diferentes tipos de controles, como (input), (textarea), (select), y botones (button) o (input type="submit"). Los formularios son fundamentales en aplicaciones web para recopilar información, como registros de usuarios, comentarios o encuestas, y pueden ser configurados para enviar datos a través de distintos métodos, como GET o POST, según el propósito de la aplicación. Además, se pueden validar tanto en el lado del cliente como en el servidor para garantizar la integridad de la información ingresada.
Etiqueta table La etiqueta (table) en HTML se utiliza para crear una tabla, que es una estructura organizada en filas y columnas para presentar datos de manera clara y accesible. Etiqueta from Se utiliza para crear un formulario, que permite a los usuarios enviar datos a un servidor.
Atributos de table Los atributos de la etiqueta "table" en HTML definen aspectos visuales y estructurales de la tabla. Los más comunes son border, que especifica el ancho del borde; cellpadding y cellspacing, que determinan el espacio interno de las celdas y el espacio entre ellas, respectivamente; width y height, que establecen dimensiones; y summary, que proporciona una descripción accesible. También se puede usar caption para añadir un título a la tabla. Muchos de estos atributos están obsoletos en HTML5, y se recomienda utilizar CSS para el estilizado. Etiqueta input La etiqueta "input" en HTML se utiliza para crear campos interactivos en formularios, permitiendo la entrada de datos por parte del usuario. Esta etiqueta es versátil y puede adoptar varios tipos mediante el atributo type, como text, password, checkbox, radio, file, entre otros, lo que determina el comportamiento y la apariencia del campo. Además, puede incluir atributos como name, value, placeholder, y required para mejorar la funcionalidad y la accesibilidad del formulario.
Etiqueta tr La etiqueta TR, que significa "fila de tabla", se usa para crear filas individuales dentro de una tabla. Etiqueta label Se utiliza en los formularios y añade una leyenda o etiqueta explicativa a elementos como, por ejemplo, campos de entrada, botones o casillas de verificación.
Atributo tr Su principal atributo es class, que permite aplicar estilos CSS específicos a esa fila, y id, que puede usarse para identificarla de manera única en scripts o estilos. También se pueden utilizar atributos de estilo en línea para personalizar la apariencia de la fila. Aunque no tiene tantos atributos como otras etiquetas, "tr" es esencial para estructurar el contenido de la tabla de manera ordenada y semántica. Etiqueta select Representa un control que muestra un menú de opciones.
Etiqueta td y th El td es usado para el cuerpo de tu tabla, mientras que el th son las que sirven para dar un resalto al titulo de cada columna de tu tabla (Generalmente se pone en negrita el texto por defecto). Etiqueta textarea Nos permite escribir párrafos y líneas de contenido de texto sin restricciones.
Atributos de td y th Ambas pueden incluir atributos como colspan y rowspan, que permiten que una celda se extienda a través de múltiples columnas o filas, respectivamente. También pueden utilizar atributos como scope, que define si el encabezado se aplica a una fila, columna o grupo, y class o id para aplicar estilos CSS o manipulación con scripts. Además, los atributos align y valign (obsoletos en HTML5) pueden usarse para alinear el contenido dentro de las celdas. Etiqueta fieldset, legend La etiqueta fieldset permite agrupar etiquetas input para mejorar su visualización. La etiqueta legend muestra un texto a modo de título. Ambas etiquetas se suelen utilizar conjuntamente, aunque no es obligatorio.
Tablas avanzadas Ya hemos visto que las tablas se van definiendo por filas mediante el elemento tr. Pues dentro de HTMLpodemos agrupar estas filas por funcionalidad. Etiqueta meter nos permite la creación de medidores que permitan mostrar el nivel o escala de un cierto valor.
Etiqueta thead Es utilizada para definir un bloque que contiene las celdas de encabezado de una tabla. Etiqueta progress Es una barra visual que muestre el progreso o avance de alguna tarea o cuestión, de modo que el usuario sepa que debe esperar mientras se realizan dichas tareas. También se puede utilizar para indicar el progreso de algún proceso o incluso el avance del usuario a través de la página.
Etiqueta tfoot Se utiliza para definir un pie de tabla. Etiqueta datalist Es una etiqueta de HTML5 que se utiliza para crear una lista de opciones predefinidas que se pueden utilizar en un campo de entrada.
Etiqueta tbody Se utiliza para agrupar el contenido de una tabla que representa el cuerpo de la tabla. Etiqueta summary Es usado como un widget de revelación a través del cual el usuario puede obtener información adicional

Define los elementos que componen la estructuración de una página web.

  1. Contenedor(wrapper)

    2. Es el elemento principal que envuelve todo el contenido de la página, ayudando a centrarlo y a establecer márgenes.

  2. Cabecera(header)

    3. Es la parte superior de la página que generalmente incluye el logo, el título del sitio y a menudo un menú de navegación.

  3. Contenido(content)

    4. Es el área principal donde se muestra la información relevante, como texto, imágenes y otros elementos interactivos.

  4. Menú(menu)

    5. Un conjunto de enlaces que permite a los usuarios navegar por diferentes secciones del sitio. Puede estar en la cabecera, lateral o en el pie.

  5. Pie(footer)

    6. La parte inferior de la página que generalmente incluye información adicional como derechos de autor, enlaces a políticas de privacidad y contacto.

  6. Lateral(sidebar)

    7. Una columna en uno o ambos lados del contenido principal que puede contener widgets, enlaces, anuncios o información adicional.

  7. Etiquetas(div)

    8. Elementos de contenedor en HTML que se utilizan para estructurar y agrupar el contenido de la página.

Identifica y describe los elementos que componen la estructuración semántica de una página web.

La estructuración semántica de una página web implica el uso de elementos HTML que no solo organizan el contenido, sino que también transmiten su significado. Elementos como "header", que define la cabecera de la página, y "nav", que agrupa enlaces de navegación, son esenciales para crear una estructura clara. El "main" representa el contenido principal, mientras que el "footer" contiene información adicional, como derechos de autor y enlaces relevantes. Asimismo, "article" y "section" permiten dividir el contenido en partes independientes y relacionadas, respectivamente, y el uso de "aside" ofrece información complementaria, como barras laterales o notas.

La jerarquía de encabezados es otro aspecto crucial en esta estructuración. Comenzando con "h1" para el título principal y descendiendo hasta "h6", estos encabezados organizan el contenido de manera lógica y facilitan la navegación. Este orden no solo ayuda a los usuarios a entender la relación entre diferentes secciones, sino que también es beneficioso para los motores de búsqueda, mejorando la indexación. Además, las listas ("ul" para listas no ordenadas y "ol" para listas ordenadas) y las tablas ("table", "tr", "td", "th") organizan la información de forma clara y accesible, lo que contribuye a una mejor experiencia de usuario.

Finalmente, la accesibilidad es un componente fundamental de la estructuración semántica. Atributos como "aria-*" y el uso de metadatos en la sección "head" mejoran la usabilidad de la página para personas con discapacidades y optimizan la indexación en motores de búsqueda. La implementación de microdatos y especificaciones de Schema.org añade una capa adicional de significado que facilita la comprensión del contenido por parte de los motores de búsqueda. En conjunto, todos estos elementos crean una estructura bien organizada que no solo mejora la experiencia del usuario, sino que también favorece el SEO y la accesibilidad.
